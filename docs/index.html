The Ising Model

Goal

A simple but realistic model that captures the magnetic behavior of a system

  • Spins, s, occupy a lattice of N sites
  • s_i is the spin at site i:
    • spin up: s_i = +1
    • spin down: s_i = -1
  • A spin configuration \{s_i\} is a specific assignment of spins to each of the N sites, e.g. \{s_1=-1, s_2 = +1, \dots, s_N = +1\}

The Ising Hamiltonian

  • Question: How does the Ising model capture magnetic behaviors?
  • Different spin configurations \Rightarrow different energies

The Hamiltonian

H(\{s_i\}) quantifies the energy of the system with a specific spin configuration \{s_i\}

  • Neighboring spins s_i and s_j interact with interaction strength J_{ij}1 H(\{s_i\}) = -\sum_{ij}J_{ij} s_{i}s_{j}

  1. Negative sign ensures the configuration that minimizes the energy is preferred

Phase Transitions

Phase Transitions

Phase transitions occur when the system undergoes a sudden change in its macroscopic properties as a function of temperature

Phase Transitions

  • Magnetization: m = \frac{1}{N}\sum_{i=1}^N s_i

  • Ising model phase transition occurs between:

    • Ferromagnetic phase: m\neq 0
    • Paramagnetic phase; m=0

  • T_c\approx 2.269 J/k_B

Restricted Boltzmann Machines

Goal

Learn the probability distribution P(\{s_i\}) over a set of binary spins \{s_i\}

  • Restricted Boltzmann Machines (RBMs): two-layered neural networks with no intra-layer connection
  • The interaction between visible units \{s_i\} and hidden units \{h_j\} is
    • note: b_j, w_{ij}, c_i are parameters

E = \sum_i b_j h_j + \sum_{ij} \textcolor{green}{\{s_i\}} w_{ij} \textcolor{Skyblue}{\{h_j\}} + \sum_i c_i \textcolor{green}{\{s_i\}}

Generating Ising Configurations

Algorithm

  1. input: Initial configuration \{s_i\}

  2. spin/cluster selection:

    • Metropolis: Randomly select a single spin
    • Wolff: Randomly select a seed spin and form a cluster

  3. flip:

    • Metropolis: Calculate the energy difference \textcolor{#07B875}{\Delta E}.
      • If \textcolor{#07B875}{\Delta E} \leq 0, flip
      • If \textcolor{#07B875}{\Delta E} >0, flip with probability e^{-\Delta E/kT}
    • Wolff: Add aligned neighboring spins to the cluster with probability P=e^{-2J/kT}. Flip the cluster.

  4. iterate: Repeat spin/cluster selection and flip

  5. equilibrate: Allow the system to equilibrate and start sampling

  6. return: return sampled equilibrated configurations \{s_i\}

Simulation technique

Monte Carlo algorithms under periodic boundary conditions

  • Metropolis algorithm: Randomly flip a spin
  • Critical slow down: hard to flip spins near critical temperature!
    • Wolff Algorithm - flip spin clusters

Optimal Transport

  • Idea: Quantitatively compare probability distributions

  • Given probability distributions (x,\mu) on \mathcal{X} and (y,\nu) on \mathcal{Y}, find a map \Pi(\mu, \nu) that minimizes the cost of the transport \inf_{\gamma \in \Pi(\mu, \nu)} \int_{\mathcal{X} \times \mathcal{Y}} c(x, y) , d\gamma(x, y)

Wasserstein Distance

For two probability measures \mu and \nu on the same space \mathcal{X}: W(\mu, \nu) = \inf_{\gamma \in \Pi(\mu, \nu)} \int_{\mathcal{X} \times \mathcal{X}} d(x, y) , d\gamma(x, y)
